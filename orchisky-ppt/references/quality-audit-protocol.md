# Quality Audit Protocol · 质量审计协议

> 适用阶段：Phase 7（逐页设计）→ Phase 8（质检与交付）。

---

## 一、三级关卡机制

| 关卡 | 检查对象 | 工具链行为 |
|:---:|:---|:---|
| Gate 1 | 空的`<text>`元素 | 输出 WARNING |
| Gate 2 | 单行CJK字符超过20个 | 输出 WARNING |
| Gate 3 | 自动修复`&`；检测 rgb()/hsl()颜色格式 | 自动修复/WARNING |

---

## 二、交付前人工审计

### HTML预览审查（强制）

```
审查清单：
□ 文字是否有溢出
□ 文字是否有重叠
□ 箭头是否正确显示
□ 多行文字是否正确换行
□ 颜色是否符合品牌规范
□ 转换日志是否有WARNING
```

### PPTX抄南3页检查

```
检查项：
□ 文字是否可编辑
□ 形状是否可独立选中
□ 图片是否变形
□ 中文字体是否正确
```

---

## 三、AI生成SVG时强制约束

```
1. 特殊字符：& → &amp;，< → &lt;，> → &gt;
2. 颜色格式：必须 hex，禁止 rgb()/hsl()
3. 换行方式：<tspan x="..." dy="1.4em">，禁止\n
4. 箭头画法：禁止<marker>，必须用<polygon>
5. 绝对坐标：禁止<g transform="rotate">
6. 单行长度：CJK单行不超过20字
```
