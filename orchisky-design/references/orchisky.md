## 1. 设计哲学与原则

### 1.1 德勤-Pentagram 视觉语言基因

参考 Pentagram（Giorgia Lupi）为 Deloitte Insights 操刀的品牌升级：**粗体排版 × 可见网格 × 克制色板**，传递严谨、秩序与信心。

### 1.2 三条核心原则

| 原则 | 含义 |
|------|------|
| **单一彩色** | 全套交付物只允许一个主品牌彩色（蓝 `#1070F0`），语义色（绿/橙/红）仅用于状态标记 |
| **信息先行** | 白底/浅灰底，无渐变无光晕无暗色背景；表格优先；图表直接标注数值，不用独立图例 |
| **符号双通道** | 所有状态标记必须同时用颜色 + 文字/符号，保证色弱可读 |

### 1.3 V2.0 品牌定位

- **行业**：制造业数字化 · 工业软件 · MES/ERP/WMS/EAM 实施
- **受众**：工厂管理层、IT 团队、投资方、合作伙伴
- **情感关键词**：可信赖、精密、务实、专业而不冷漠

### 2.3 语义色（必须颜色 + 符号双通道）

| 状态 | 符号 | 字色 | 背景色 |
|------|------|------|--------|
| 正常 / 通过 | ● | `#1A7A42` | `#EAF5EF` |
| 警告 / 隐患 | ▲ | `#9A5200` | `#FEF4E8` |
| 断点 / 异常 | ✕ | `#B01C1C` | `#FDEDED` |
| 规划中 / 信息 | → | `#1070F0` | `#EAF2FE` |
| 待定 / 禁用 | ○ | `#888888` | `#F5F5F5` |

### 2.4 图表系列色顺序

```

正向：#1A7A42（绿）

```

## 4. Logo 与品牌基础

### 4.1 Logo 规则

- **标准组合**：图形符号 + 中文全称（兰之天）+ 英文副标题（Orchisky）
- **最小尺寸**：印刷 16mm 宽；屏幕 64px 宽
- **安全区**：Logo 宽度的 1/4 作为全周留白
- **背景规则**：白底（首选）/ 深蓝底（`blue-700`）/ 浅蓝底（`blue-50`）

### 4.2 Logo 禁用清单

- ❌ 修改比例或间距
- ❌ 放置在彩色或图片底上（除非专项设计版）
- ❌ 加投影、发光、渐变
- ❌ 仅使用中文或仅使用英文（须组合使用）

## 5. 图标系统

| 项目 | 规格 |
|------|------|
| 图标库 | Lucide Icons（开源，线性风格） |
| 网格尺寸 | 24×24px 基准 |
| 描边宽度 | 2px（UI 场景）/ 1.5px（密集信息） |
| 风格 | 纯线性（Outlined），**不使用 Filled** |
| 颜色 | 继承父元素文字色，交互态用 `blue-500` |

禁用：填充彩色块、多色渐变、3D 拟物、Emoji 代替、线性+填充混用。

## 6. 插画规范

- **类型**：线框插画（Outline Illustration），非卡通、非照片
- **线条**：2px 等宽描边，`ink-600` 或 `blue-500`
- **配色**：仅允许 `blue-50`/`ink-100` 作为填充底色，无多色渐变
- **使用场景**：空状态、登录页装饰区、品牌物料封面

## 7. 通用组件库

### 7.1 表格

| 元素 | 规格 |
|------|------|
| 表头底色 | `ink-800 #1A1A1A`，思源黑体 Bold 9pt 白字 |
| 表头下边线 | 1pt `blue-500 #1070F0` |
| 数据行 | 思源宋体 9pt，奇行白 `ink-0` / 偶行 `ink-50` |
| 内横线 | 0.5pt `ink-200 #E2E2E2` |
| **竖线** | **全部取消** |
| 末行底线 | 1pt `ink-600 #555555` |
| 表名位置 | 表格**下方**居中，斜体 8.5pt `ink-500` |

### 7.2 Callout 提示框（4 类）

| 类型 | 底色 | 左边条 | 标签色 |
|------|------|--------|--------|
| 信息型 | `blue-50` | `blue-500` | `blue-600` |
| 警告型 | `#FEF4E8` | `#9A5200` | `#9A5200` |
| 正向型 | `#EAF5EF` | `#1A7A42` | `#1A7A42` |
| 待决策型 | `ink-100` | `ink-600` | `ink-800` |

### 7.3 KPI 指标卡

- 底色 `ink-100`，圆角 **2px**，**无阴影**
- 顶部 3pt 色条：蓝=计划/正向 / 橙=警告 / 红=问题 / 绿=达标
- 数字使用 JetBrains Mono

### 7.4 按钮

| 类型 | 底色 | 字色 | 圆角 |
|------|------|------|------|
| Primary | `blue-500` | `ink-0` | **4px** |
| Default | `ink-0` | `ink-800`，边 1px `ink-300` | 4px |
| Danger（实填） | `#B01C1C` | `ink-0` | 4px |
| Danger（描边） | `ink-0` | `#B01C1C`，边 `#B01C1C` | 4px |
| Ghost | 透明 | `ink-600` | 4px |

> **圆角约定**：Tag/Badge = 2px；按钮/输入框 = 4px；卡片/Modal = 6px；**禁用胶囊（999px）**。

## 12. 品牌音调

| 维度 | 描述 |
|------|------|
| 专业而不冷漠 | 精准术语，避免堆砌缩写黑话 |
| 务实而不保守 | 直接说结论，先讲"做什么"再讲"为什么" |
| 数据驱动 | 具体数字优先，避免"显著""大幅""明显" |
| 制造业语境 | 工厂现场视角，不用互联网黑话 |

**禁用词**：赋能、生态、闭环、底层逻辑、降维打击

## 13. 禁用规则（Hard Rules）

| 禁止内容 | 适用范围 |
|---------|----------|
| 第二个装饰彩色 | 全套 |
| 渐变 / 光晕 / 投影 / 3D 效果 | 全套 |
| 表格竖线 | Word / Excel / PPT |
| 单独依赖颜色传达状态（无符号） | 全套 |
| 纯黑 `#000000` | 全套 |
| 超过 2 种字体（等宽字体除外） | 全套 |
| PPT 主题词式 Action Title | PPT |
| 图表独立图例 | 全套 |
| 饼图 / 3D 爆炸效果 | 全套 |
| 数据无来源标注 | 全套 |
| 圆角 > 6px（UI） | UI |
| 胶囊形按钮（999px） | UI |
| Emoji 用于正式文档 | 全套 |
