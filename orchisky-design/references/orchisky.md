## 1. 设计原则

### 1.1 单一强调色原则

整套规范只允许**一个彩色强调色**（深蓝 `#003D7A`），其余全部为近黑/白/灰中性色序列。语义色（绿/橙/红）仅用于状态标记，不作装饰。

### 1.2 信息密度原则

- 白底/浅灰底，无渐变、无光晕、无暗色背景
- 表格优先，能用表格表达的不用文字堆砌
- 高信息密度，但每张幻灯片只传达一个核心结论
- 图表直接标注数值，不使用独立图例

### 1.3 德勤视觉语言核心

参考 Pentagram（Giorgia Lupi）为 Deloitte Insights 操刀的设计语言：新的可视化语言以**粗体排版和可见网格系统**为特征，传递严谨、秩序与信心。

## 2. 配色体系

### 2.1 主品牌色

| 色值 | 名称 | 用途 |
|------|------|------|
| `#1A1A1A` | 近黑 | 标题、表头底色、Action Title 区 |
| `#003D7A` | 品牌深蓝 ★ | **全套唯一强调色** |
| `#0056A8` | 次级蓝 | H2 标题、PPT 章节色、图表系列 2 |
| `#3A7FC1` | 中蓝 | 图表系列 3 |
| `#E8F0FA` | 浅蓝底 | Callout 底色、PPT 卡片底 |

### 2.2 中性色序列

| 色值 | 名称 | 用途 |
|------|------|------|
| `#333333` | 正文灰 | 正文、表格数据 |
| `#555555` | 辅助灰 | 副标题、说明文字 |
| `#888888` | 注释灰 | 图注、页脚 |
| `#CCCCCC` | H2 底线 | H2 分隔线 |
| `#E2E2E2` | 表格横线 | 表格横线、边框 |
| `#F5F5F5` | 背景灰 | 卡片底色 |
| `#FFFFFF` | 纯白 | 页面底色 |

### 2.3 语义色（必须符号+颜色双通道）

| 符号 | 含义 | 字色 | 背景色 |
|------|------|------|------|
| ● | 正常 / 通过 | `#1A7A42` | `#EAF5EF` |
| ▲ | 警告 / 隐患 | `#9A5200` | `#FEF4E8` |
| ✕ | 断点 / 异常 | `#B01C1C` | `#FDEDED` |
| → | 规划中 | `#003D7A` | `#E8F0FA` |
| ○ | 待定 | `#888888` | `#F5F5F5` |

### 2.4 图表系列色顺序

`#003D7A` → `#0056A8` → `#3A7FC1` → `#A8C8ED` | 对比/灰 `#888888` | 正向 `#1A7A42` | 负向 `#B01C1C`

## 3. 字体体系

### 3.1 两字体原则

**微软雅黑（无衬线）+ 宋体-简（衬线）**，全套不超过两种字体。

| 字体 | 角色 | 跨平台备选 |
|------|------|----------|
| **微软雅黑** Microsoft YaHei | 标题 / UI / 表头 / 页眉页脚 | macOS：Heiti SC 黑体-简 |
| **宋体-简** Songti SC | 正文段落 / 表格数据行 | Windows：SimSun 宋体 |

### 3.2 字号规范

| 层级 | 字体 | 字号 | 颜色 |
|------|------|------|------|
| H1 章标题 | 微软雅黑 Bold | 22pt | `#1A1A1A` |
| H2 节标题 | 微软雅黑 Bold | 15pt | `#1A1A1A` |
| H3 小节 | 微软雅黑 Bold | 12pt | `#555555` |
| 正文 Body | 宋体-简 Regular | 10.5pt | `#333333` |
| 表格数据 | 宋体-简 Regular | 9pt | `#333333` |
| Callout | 微软雅黑 Regular | 9.5pt | `#555555` |
| 表名/图注 | 微软雅黑 Regular | 8.5pt | `#888888` |
| 页眉/页脚 | 微软雅黑 Regular | 8pt | `#888888` |
| PPT Action Title | 宋体-简 Bold | 18–22pt | `#1A1A1A` |

## 4. 通用组件库

### 4.1 表格（核心组件）

| 元素 | 规格 |
|------|------|
| 表头底色 | `#1A1A1A`，微软雅黑 Bold 9pt 白字 |
| 表头下边线 | 1pt `#003D7A` |
| 数据行 | 宋体-简 9pt，奇行白 / 偶行 `#FAFAFA` |
| 内横线 | 0.5pt `#E2E2E2` |
| **竖线** | **全部取消** |
| 末行底线 | 1pt `#555555` |
| 表名位置 | 表格**下方**居中，斜体 8.5pt 灰色 |
| 行高 | 表头 0.75cm，数据行 0.65cm |

### 4.2 Callout 提示框（4 类）

| 类型 | 底色 | 标签颜色 |
|------|------|----------|
| 信息型 | `#E8F0FA` | `#003D7A` |
| 警告型 | `#FEF4E8` | `#9A5200` |
| 正向型 | `#EAF5EF` | `#1A7A42` |
| 待决策型 | `#F5F5F5` | `#1A1A1A` |

### 4.3 KPI 指标卡

- 底色 `#F5F5F5`，圆角 3px
- 顶部 3pt 色条：红=问题 / 橙=警告 / 蓝=计划 / 绿=达标

### 7.1 基础参数

| 参数 | 规格 |
|------|------|
| 演示尺寸 | 33.87 × 19.05cm（16:9） |

| Action Title 区高度 | 约 2.2cm |

### 7.2 幻灯片解剖层次（标准 Chrome）

```
┌──────────────────────────────────────────────────────┐
│  ░░░░░░░░░░░░ 3pt 品牌蓝顶线 #003D7A ░░░░░░░░░░░░░░ │
│  Section路径 · 子级路径           ORCHISKY  │  ← 页眉
│ ‖                                                     │
│ ‖  ACTION TITLE: 宋体-简 Bold 18–22pt #1A1A1A        │
│ ‖  完整结论句，全页最大字号                            │
│  ─────────────────────────────────────────── 0.5pt   │
│              内 容 区                                  │
│  ─────────────────────────────────────────── 0.5pt   │
│  来源：Orchisky 调研整理 · 2026-04          第 X 页  │
└──────────────────────────────────────────────────────┘
```

**注意**：Action Title 左侧的 `‖` 表示 3pt 品牌蓝竖线（`#003D7A`）。

### 7.3 Action Title 黄金规则

- ✕ **错误**：`计划现状分析`（主题词式）
- ✓ **正确**：`11个计划环节5个断点集中在中游，MES接管可消除重录与延迟`

## 9. 禁用规则

| 禁止内容 | 适用范围 |
|---------|----------|
| 第二个装饰彩色 | 全套 |
| 渐变 / 光晕 / 投影 / 3D 效果 | 全套 |
| 表格竖线 | Word / Excel |
| 单独依赖颜色传达状态 | 全套 |

## 10. 交付前质量清单

- [ ] 封面信息完整（标题、委托方、版本号、日期、保密标记）

- [ ] 交付型：表头使用黑底/深蓝底样式
- [ ] 工作型：输入单元格已标蓝色字体
- [ ] Sheet 结构规范（Cover → Summary → Detail → Assumptions）

- [ ] 每页 Action Title 是完整结论句而非主题词
- [ ] Action Title 使用白底深色文字 + 左侧品牌蓝竖线

- [ ] 未使用第二个装饰彩色
- [ ] 所有状态标记使用符号 + 颜色双通道
