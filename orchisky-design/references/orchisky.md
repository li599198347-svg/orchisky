## 1. 设计原则

### 1.1 单一强调色原则

整套规范只允许**一个彩色强调色**（深蓝 `#003D7A`），其余全部为近黑/白/灰中性色序列。语义色（绿/橙/红）仅用于状态标记，不作装饰。

Word、Excel、PPT 三套文档的配色、字体、组件样式**完全统一**，客户在任何交付物中看到的是同一套视觉语言。

### 1.2 信息密度原则

- 白底/浅灰底，无渐变、无光晕、无暗色背景
- 表格优先，能用表格表达的不用文字堆砌
- 高信息密度，但每张幻灯片只传达一个核心结论
- 图表直接标注数值，不使用独立图例

### 1.3 德勤视觉语言核心

参考 Pentagram（Giorgia Lupi）为 Deloitte Insights 操刀的设计语言：

> "新的可视化语言以**粗体排版和可见网格系统**为特征（灵感来自税务账簿），传递严谨、秩序与信心。克制的色板让单一强调色突出，确保读者不会被多重信息淹没。"

## 2. 配色体系

### 2.1 主品牌色

| 色值 | 名称 | 用途 |
|------|------|------|
| `#1A1A1A` | 近黑 | 标题、表头底色、Action Title 区 |
| `#003D7A` | 品牌深蓝 ★ | **全套唯一强调色** |
| `#0056A8` | 次级蓝 | H2 标题、PPT 章节色、图表系列 2 |
| `#3A7FC1` | 中蓝 | 图表系列 3 |
| `#E8F0FA` | 浅蓝底 | Callout 底色、PPT 卡片底 |

### 2.2 中性色序列

| 色值 | 名称 | 用途 |
|------|------|------|
| `#333333` | 正文灰 | 正文、表格数据 |
| `#555555` | 辅助灰 | 副标题、说明文字 |
| `#888888` | 注释灰 | 图注、页脚 |
| `#CCCCCC` | H2 底线 | H2 分隔线 |
| `#E2E2E2` | 表格横线 | 表格横线、边框 |
| `#F5F5F5` | 背景灰 | 卡片底色、奇偶行 |
| `#FFFFFF` | 纯白 | 页面底色 |

### 2.3 语义色（必须符号+颜色双通道）

| 符号 | 含义 | 字色 | 背景色 |
|------|------|------|------|
| ● | 正常 / 通过 | `#1A7A42` | `#EAF5EF` |
| ▲ | 警告 / 隐患 | `#9A5200` | `#FEF4E8` |
| ✕ | 断点 / 异常 | `#B01C1C` | `#FDEDED` |
| → | 规划中 | `#003D7A` | `#E8F0FA` |
| ○ | 待定 | `#888888` | `#F5F5F5` |

> **重要**：禁止单独依赖颜色传达状态——必须同时使用符号作为第二通道，确保色觉障碍用户可识别。

### 2.4 图表系列色顺序

`#003D7A` → `#0056A8` → `#3A7FC1` → `#A8C8ED` | 对比/灰 `#888888` | 正向 `#1A7A42` | 负向 `#B01C1C`

图表规则：纯色填充，不用渐变；无图表边框；背景白色；网格线 0.5pt `#E2E2E2`；直接在数据点标注数值，不设独立图例。

## 3. 字体体系

### 3.1 两字体原则

**微软雅黑（无衬线）+ 宋体-简（衬线）**，全套不超过两种字体。

| 字体 | 角色 | 跨平台备选 |
|------|------|-----------|
| **微软雅黑** Microsoft YaHei | 标题 / UI / 表头 / 页眉页脚 / Callout / 状态标签 | macOS 无 Office：Heiti SC 黑体-简 |
| **宋体-简** Songti SC | 正文段落 / 表格数据行 | Windows：SimSun 宋体 |

> 英文/数字在正文中随中文字体走；数字右对齐时可用等宽字体 `Courier New` 提升对齐精度。

### 3.2 字号规范

| 层级 | 字体 | 字号 | 颜色 | 其他 |
|------|------|------|------|------|
| H1 章标题 | 微软雅黑 Bold | 22pt | `#1A1A1A` | 段前 24pt |
| H2 节标题 | 微软雅黑 Bold | 15pt | `#1A1A1A` | 底线 0.5pt `#CCCCCC` |
| H3 小节 | 微软雅黑 Bold | 12pt | `#555555` | 段前 11pt |
| 正文 Body | 宋体-简 Regular | 10.5pt | `#333333` | 行距 1.4，首行缩进 2 字 |
| 表格数据 | 宋体-简 Regular | 9pt | `#333333` | — |
| Callout | 微软雅黑 Regular | 9.5pt | `#555555` | 底色 `#E8F0FA` |
| 表名/图注 | 微软雅黑 Regular | 8.5pt | `#888888` | 斜体，表格下方居中 |
| 页眉/页脚 | 微软雅黑 Regular | 8pt | `#888888` | — |
| PPT Action Title | 宋体-简 Bold | 18–22pt | `#1A1A1A` | 见 PPT 规范 |

## 4. 通用组件库

### 4.1 表格（核心组件）

| 元素 | 规格 |
|------|------|
| 表头底色 | `#1A1A1A`，微软雅黑 Bold 9pt 白字 |
| 表头下边线 | 1pt `#003D7A` |
| 数据行 | 宋体-简 9pt，奇行白 / 偶行 `#FAFAFA` |
| 内横线 | 0.5pt `#E2E2E2` |
| **竖线** | **全部取消** |
| 末行底线 | 1pt `#555555` |
| 表名位置 | 表格**下方**居中，斜体 8.5pt 灰色 |
| 引导语 | 每张表格前必须有 1–2 句引导语 |
| 行高 | 表头 0.75cm，数据行 0.65cm |

### 4.2 Callout 提示框（4 类）

| 类型 | 底色 | 标签颜色 | 用途 |
|------|------|---------|------|
| 信息型 | `#E8F0FA` | `#003D7A` | 客观发现、关键说明 |
| 警告型 | `#FEF4E8` | `#9A5200` | 风险、隐患提醒 |
| 正向型 | `#EAF5EF` | `#1A7A42` | 里程碑确认、正面结论 |
| 待决策型 | `#F5F5F5` | `#1A1A1A` | 开放项、待跟进事项 |

### 4.3 KPI 指标卡

- 底色 `#F5F5F5`，圆角 3px
- 顶部 3pt 色条：红=问题 / 橙=警告 / 蓝=计划 / 绿=达标
- 指标数字：微软雅黑 Bold，颜色与顶部色条一致
- 描述文字：微软雅黑 Regular 9pt 灰色

## 6. Excel 工作簿规范

### 6.1 两类 Excel

**交付型 Excel**（报价单、评分表——客户直接查看）遵循与 Word/PPT 一致的配色规范。

**工作型 Excel**（数据分析、ROI 测算——内部使用）遵循**投行业界字色编码惯例**。

### 6.2 交付型：单元格样式体系

| 样式名 | 底色 | 字体 | 用途 |
|--------|------|------|------|
| Header 表头 | `#1A1A1A` | 微软雅黑 Bold 9pt 白 | 列标题行 |
| Sub-Header | `#003D7A` | 微软雅黑 Bold 9pt 白 | 分节标题行 |
| Data 奇行 | `#FFFFFF` | 宋体-简 9pt `#333` | 普通数据 |
| Data 偶行 | `#FAFAFA` | 宋体-简 9pt `#333` | 间隔底色 |
| Sum 汇总行 | `#E8F0FA` | 微软雅黑 Bold `#003D7A` | 合计/小计 |
| OK 正常 | `#EAF5EF` | 微软雅黑 Bold `#1A7A42` | 状态标记 |
| Warning 警告 | `#FEF4E8` | 微软雅黑 Bold `#9A5200` | 状态标记 |
| Danger 断点 | `#FDEDED` | 微软雅黑 Bold `#B01C1C` | 状态标记 |

### 6.3 工作型：字色编码体系

| 字体颜色 | 含义 | 使用场景 |
|---------|------|---------|
| 蓝色 `#0056A8` | 硬编码输入 / 假设 | 历史数据、人工输入的参数 |
| 黑色 `#1A1A1A` | 本表公式 / 计算结果 | 在本工作表内计算的所有公式 |
| 绿色 `#1A7A42` | 跨工作表引用 | 引用同文件其他 Sheet 的数据 |
| 红色 `#B01C1C` | 外部文件链接 | 引用其他 Excel 文件（慎用） |

### 6.4 数字格式规范

| 数据类型 | 格式 | 对齐 |
|---------|------|------|
| 金额/报价 | `¥#,##0.00` | 右对齐 |
| 百分比 | `0.0%` | 右对齐 |
| 整数计数 | `#,##0` | 右对齐 |
| 负数 | `(1,234)` 括号格式 | 右对齐 |
| 日期 | `YYYY-MM-DD` | 居中 |
| 状态文字 | 配合语义色底色 | 居中 |

### 6.5 工作簿 Sheet 结构规范

| 位置 | Sheet 名 | 标签颜色 | 内容 |
|------|---------|---------|------|
| 第 1 张 | 封面 Cover | 黑色 | 文档标题、版本、使用说明 |
| 第 2 张 | 总览 Summary | 深蓝 | 关键指标汇总仪表板 |
| 3–N 张 | 明细 Detail_XXX | 浅蓝 | 各模块数据，按章节命名 |
| 最后一张 | 假设 Assumptions | 橙色 | 所有蓝色输入参数集中管理 |

### 7.2 字号层级

| 层级 | 字体 | 字号 | 颜色 |
|------|------|------|------|
| 封面主标题 | 宋体-简 Bold | 32–40pt | `#FFFFFF` |
| Action Title | 宋体-简 Bold | 18–22pt | `#1A1A1A` |
| 内容小标题 | 微软雅黑 Bold | 13–15pt | `#003D7A` |
| 正文说明 | 微软雅黑 Regular | 11–12pt | `#555555` |
| 图注/来源 | 微软雅黑 Regular | 8–9pt | `#888888` |

### 7.3 幻灯片解剖层次（标准 Chrome）

```
┌──────────────────────────────────────────────────────┐
│  ░░░░░░░░░░░░ 3pt 品牌蓝顶线 #003D7A ░░░░░░░░░░░░░░ │
│  Section路径 · 子级路径           ORCHISKY  │  ← 页眉 0.6cm
│  ─────────────────────────────────────────── 0.5pt   │
│ ‖                                                     │
│ ‖  ACTION TITLE: 宋体-简 Bold 18–22pt #1A1A1A        │  ← AT区 ~2.2cm
│ ‖  完整结论句，全页最大字号，不使用色块包裹             │
│  ─────────────────────────────────────────── 0.5pt   │
│                                                       │
│              内 容 区（23 种版式在此变化）              │
│                                                       │
│  ─────────────────────────────────────────── 0.5pt   │
│  来源：Orchisky 调研整理 · 2026-04          第 X 页  │  ← 页脚 0.5cm
└──────────────────────────────────────────────────────┘
```

**注意**：Action Title 左侧的 `‖` 表示 3pt 品牌蓝竖线（`#003D7A`），这是德勤体系与麦肯锡黑底白字 AT 最重要的视觉区别。

### 7.4 Action Title 黄金规则

把幻灯片倒过来，只读所有 AT，应能理解完整的论证逻辑链。

- ✕ **错误**：`计划现状分析`（主题词式，不传达信息）
- ✓ **正确**：`11个计划环节5个断点集中在中游，MES接管可消除重录与延迟`（完整结论句）

## 9. 禁用规则

以下规则适用于全套交付物，无例外，不因场景豁免。

| 禁止内容 | 原因 | 适用范围 |
|---------|------|---------|
| 第二个装饰彩色 | 违反单一强调色原则，信噪比上升 | 全套 |
| 渐变 / 光晕 / 投影 / 3D 效果 | AI 感强，不符合咨询专业风格 | 全套 |
