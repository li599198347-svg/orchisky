# Orchisky 设计系统 — 轨道分流与统一规则

> **核心结论**：四份文档**不是相互矛盾**，而是**面向不同载体的"分册"**。冲突来自混用。本文件把冲突点显性化，并给出"用哪套"的判断树。

---

## 0. 三张表：四份文档 × 载体 × 主色

| 文档 | 主要载体 | 主色 | 圆角基准 |
|------|---------|------|----------|
| **Orchisky Design V2.0** | Word/Excel/PPT 交付物 | `#1070F0` blue-500 | —（文档，无 px 圆角） |
| **Field Design System V1.0** | PDA / 工业 Pad HTML | `#1070F0` blue-500 | 2 / 4 / 6 px |
| **原型设计规范 V1.0（品牌版）** | 独立品牌 Demo HTML | `#1070F0` blue-500 | 2 / 4 / 6 px |
| **原型设计规范 V1.0（平台版）** | Orchisky 平台二开 HTML | `#1677FF`（Ant 5 原色） | 4 / 6 / 8 px |

> **三份文档共享同一主色 `#1070F0`。平台版因继承 Ant Design 5 使用 `#1677FF`，这是唯一合理的主色差异。**

## 2. 主色冲突（最高频）

**问题**：我看到 `#1677FF` 和 `#1070F0` 两个蓝，哪个对？

| 场景 | 正确主色 | 原因 |
|------|---------|------|
| Word / PPT / Excel | `#1070F0` | V2.0 品牌主色 |
| Field PDA/Pad HTML | `#1070F0` | 继承 V2.0 品牌色 |
| 品牌版 Demo HTML | `#1070F0` | 继承 V2.0 品牌色 |
| 平台版 Demo HTML | `#1677FF` | Ant Design 5 colorPrimary，不覆盖 |

**结论**：除平台版外，一律用 `#1070F0`。`#1677FF` **只在平台版出现**，两者不混用。

---

## 3. 圆角冲突

**问题**：按钮圆角到底是 4px 还是 6px？

| 场景 | 按钮圆角 | Tag 圆角 | 卡片/Modal |
|------|---------|---------|------------|
| V2.0 文档（PPT 形状等） | 不适用（文档形状） | — | — |
| Field / 品牌版 HTML | **4px** | 2px | 6px |
| 平台版 HTML | **6px**（Ant 5 默认） | 4px | 8px |

**结论**：Field/品牌版用 2/4/6；平台版用 4/6/8。两者不混用。

---

## 4. 字体冲突

**问题**：正文用思源黑体还是微软雅黑？

- V2.0 **首选** 思源黑体/思源宋体，兜底微软雅黑/宋体-简（Windows 无需安装额外字体）
- Field / 品牌版 HTML：Google Fonts 加载 Source Han Sans SC + Roboto Mono
- 平台版 HTML：Noto Sans SC + JetBrains Mono（Ant 5 字体栈）
- **三者结果相近，均为黑体+等宽，不冲突**

## 6. 状态 Chip 格式

| 场景 | 格式 |
|------|------|
| PPT / Word / Excel | 符号 + 颜色双通道（● ▲ ✕）|
| Field / 品牌版 HTML | CSS Chip：浅底（50色阶）+ 深字（600色阶）+ 文字语义 |
| 平台版 HTML | Ant Tag 实色填充（已审核=蓝填 / 已完成=绿填 等）|

---

## 7. 禁用规则的共同子集

无论哪份文档，以下规则**全部适用**：

- ❌ 第二个装饰彩色
- ❌ 渐变 / 3D / 投影
- ❌ 纯黑 `#000000`
- ❌ 单独依赖颜色传达状态（无符号/文字）
- ❌ 饼图 / 独立图例
- ❌ Emoji 用于正式内容
