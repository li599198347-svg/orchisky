# Orchisky 设计系统 — 轨道分流与统一规则

> **核心结论**：四份文档**不是相互矛盾**，而是**面向不同载体的"分册"**。冲突来自混用。本文件把冲突点显性化，并给出"用哪套"的判断树。

---

## 0. 三条根本原则（跨所有轨道）

无论哪个轨道，下列三条永不动摇：

1. **单一彩色**：每件作品只有一个强调色，其余皆中性灰 + 语义色。
2. **状态双通道**：颜色 + 符号/文字，永不仅凭颜色传达语义。
3. **禁用纯黑 `#000000`**：最深使用 `--ink-800 #1A1A1A`。

## 2. 主色冲突解析

| 场景 | 正确主色 | 来源 |
|------|---------|------|
| Word / PPT / Excel | `#1070F0` | V2.0 品牌规范 |
| Field PDA/Pad HTML | `#1070F0` | Field V1.0 继承 V2.0 |
| 品牌版 Demo HTML | `#1070F0` | 品牌版规范 V1.0 |
| 平台版 Demo HTML | `#1677FF` | Ant Design 5 colorPrimary |

**结论**：除平台版外，一律 `#1070F0`。`#1677FF` **只在平台版出现**，两者不混用。

---

## 4. 字体冲突解析

全部场景首选思源黑体/思源宋体，Windows 兜底微软雅黑/宋体-简，等宽用 JetBrains Mono / Roboto Mono。三者字形相近，不冲突。

---

## 5. 共同禁用规则（所有轨道）

- ❌ 第二个装饰彩色
- ❌ 渐变 / 光晕 / 投影 / 3D 效果
- ❌ 表格竖线
- ❌ 纯黑 `#000000`
- ❌ 饼图 / 独立图例
- ❌ Emoji 用于正式内容
- ❌ 仅凭颜色传达状态（无符号/文字）
