# Prompt 起手模板 — Orchisky 平台版

> 配套文档：《原型设计规范 - Orchisky 平台版 V1.0》
>
> 用途：基于 Orchisky 平台的客户 Demo 起手 Prompt，仅覆盖 **PC 后台管理系统** 场景。
>
> 填空约定：`<xxx>` 需替换，`【可选】` 段落按需删除。

---

## 0. 三条通用前置（所有 Prompt 必加）

```
基于 Orchisky 平台（Ant Design 5 定制）开发此原型，严格遵循以下视觉约束：

1. 基础组件与样式基于 Ant Design 5，所有色值 / 圆角 / 间距 / 字重延续 Ant 5 默认 token。
2. 主色 #1677FF，圆角默认 6px（Tag 4px · Modal 8px），控件默认高 32px。
3. 顶栏 48px 深色 #001529 白字，激活 tab 白底蓝字；侧栏白底，激活项 #E6F4FF 药丸 + 蓝字 + 500 字重。
4. 状态标签一律用实色填充小药丸（已审核=蓝填 · 已同步=绿填 · 已驳回=红填 · 冻结=灰填）。
5. 字体：中文 Noto Sans SC / PingFang SC，数字编码 JetBrains Mono。
6. 表头背景 #FAFAFA，主键列 #1677FF 等宽字链接，行高 44px，hover 背景 #F5F5F5。
7. 表单多列网格（3-4 列），标签右对齐 + 冒号，必填红 * 在标签前，段标题 600 · 15px · 无图标无装饰。
8. 禁止：卡片阴影、彩色表头、侧栏彩色、渐变按钮、emoji、3D 图标、圆角 12px+、标签上浮。
9. 假数据参考真实 ERP 业务（物料编码如 M-BRG-A2024-02 / 供应商如"苏州精密机械有限公司"）。
10. 只输出单页 HTML，内联样式用 CSS 变量实现 Ant token，不引入外部 React / Ant 库。
```

## 1. LIST · 列表页模板

### 1.1 使用场景

物料列表、采购入库单列表、出库单列表、库存查询、客户档案、供应商管理、工单列表

### 1.2 完整 Prompt

```
【三条通用前置】

请生成一个 Orchisky 平台的【<模块名>】列表页，结构如下：

【整体布局】
- 顶栏 48px，深色 #001529，左置 Orchisky Logo，一级 Tab 激活态为当前模块，右侧图标区。
- 侧栏 220px，白底，当前模块菜单高亮。
- 主内容区背景 #FAFAFA。

【筛选卡片】（白卡 · 6px 圆角 · 1px #F0F0F0 边）
一行放 3 个三段式筛选条件，底部按钮：[提交]（主蓝）[重置]（默认）[+ 添加条件]（链接按钮）

【表格卡片】
工具栏：左侧筛选入口，右侧批操作 + 导出。
表格列：☑ | 行号 | 编码（主键·蓝色等宽链接）| 业务字段… | 状态（实填Tag）| 操作
至少 10 行假数据，状态分布真实（70% 正常 / 20% 待处理 / 10% 异常）。
底部分页：右对齐，"共 N 条" + 页码 + "10 条/页" 下拉。
```

## 2. DETAIL · 详情页模板

```
【三条通用前置】

请生成一个 Orchisky 平台的【<模块名>详情】页：

- 分段信息块（基础信息 / 单位与包装 / 库存控制），4 列网格，标签右对齐加冒号。
- 关联子表（新的白卡），5-8 行数据。
- 底部操作栏：[编辑]（主蓝）[打印][导出][返回]（Text）

段落标题：加粗 600 · 15px · #262626 · 无图标 · 无装饰
```

## 3. CREATE · 新增/编辑页模板

```
【三条通用前置】

请生成一个 Orchisky 平台的【<模块名>新增】页：

- 分段表单（基础信息 + 可选分类段落），4 列网格，标签右对齐，必填红 * 在标签前。
- 明细子表：可编辑表格，初始空态居中灰字"请添加或选择数据"。
- 底部操作栏（左对齐）：[提交]（主蓝）[保存草稿]（默认）[取消]（默认）
```
